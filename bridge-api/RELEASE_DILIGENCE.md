# Release and Diligence Evidence

For every pilot release preserve:

- Git commit and tag for backend and mobile.
- Backend test, secret scan, dependency audit, and deployed health output.
- Expo Doctor, production verification, Android export, EAS build ID, and APK
  checksum.
- Generated SBOM and third-party license inventory.
- Render and EAS deployment/build logs.
- Database migration record and backup confirmation.
- Dataset source URLs, dates, checksums, query scope, and attribution.
- Google key-restriction screenshots and written permission records.
- AI model/configuration record and approved data-handling policy.

Update dependency notices and regenerate compliance artifacts before each
release. Keep evidence in an access-controlled release folder.
